Settings

Important: Settings configured here are saved to /data/db/settings.json and will override any corresponding environment variables. To revert to environment variable values, click the "Reset" button for each setting, or use "Reset All" to clear all overrides.

Note: Settings are applied immediately after saving.
Auto Update Settings
Determines the default behavior for stack updates when no specific policy is set in the compose file. Can be overridden per-stack using #UpdatePolicy=... comment.
Example: AutoUpdate
Schedule for automatic update checks using standard CRON syntax. Determines how often Whale Mate checks for available updates.
Examples: */5 * * * * (every 5 min), 0 */6 * * * (every 6 hours), 0 2 * * * (daily at 2am)
Portainer Settings
The URL of your Portainer instance. Required for Portainer stack management features.
Example: https://portainer.example.com or http://192.168.1.100:9000
Your Portainer API access token. Generate in Portainer under My Account > Access Tokens.
Format: ptr_xxxxxxxxxxxxx
NTFY Notifications
Enable or disable push notifications via NTFY for stack updates and other events.
The full URL of your NTFY topic where notifications will be sent.
Examples: https://ntfy.sh/myserver, https://ntfy.yourdomain.com/stack-updates
Authentication token for NTFY. Required only if your NTFY instance has access control enabled.
Only needed for private/secured NTFY instances
Test Configuration

Send a test notification to verify your NTFY configuration using the current settings. This works regardless of whether NTFY is enabled.
